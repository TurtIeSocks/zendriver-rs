Expand description

Top-level error type returned by every fallible API in this crate.

#[non_exhaustive] — new variants may be added in minor releases. Pattern-match defensively (use a _ arm).

§Examples

let e = ZendriverError::Timeout(Duration::from_secs(5));
assert_eq!(e.to_string(), "timed out after 5s");

Variants (Non-exhaustive)§

This enum is marked as non-exhaustive
Non-exhaustive enums could have additional variants added in future. Therefore, when matching against variants of non-exhaustive enums, an extra wildcard arm must be added to account for any future variants.
§

Browser(BrowserError)

Chrome process / launch failure.

§

Transport(Box<TransportError>)

Lower-level transport (WebSocket) failure.

§

Disconnected

The connection to Chrome dropped unexpectedly — the WebSocket died without a caller-requested Browser::close (Chrome crashed, the socket was severed, a Close frame arrived out-of-band). A CDP call awaiting a reply when the drop happened resolves to this, distinct from the clean-shutdown ZendriverError::Transport surfaced after close().

Recover with Browser::reconnect (which re-dials the live Chrome process) — but note that existing Tab/Frame/Element handles are invalidated by a reconnect and must be re-acquired via Browser::main_tab / Browser::tabs.

§

EventStreamIncomplete

An awaited event stream ended — or lost delivery continuity — before the condition being watched for could be observed. Distinct from a timeout: the awaited condition may in fact have occurred, but a transport teardown, reconnect, or dropped broadcast subscriber means we can no longer prove whether it did, so reporting a plain timeout (or a complete result) would be a lie. Callers that need a definitive answer should re-establish the wait — after a Browser::reconnect if the connection itself dropped.

§

Cdp

Chrome returned a CDP RPC error (a method call returned error.code / error.message).

Fields

§code: i32

CDP RPC error code (typically negative; see JSON-RPC spec).

§message: String

Human-readable error message from Chrome.

§data: Option<Value>

Optional data field from the RPC error payload.

§

ElementNotFound

A query selector did not match within the timeout.

Fields

§selector: String

Description of the selector that failed to match (e.g. "css(button.primary)", "text_exact(Submit)").

§

ConflictingSelectors

A predicate method (.tag/.attr/.containing_text/…) was combined with a single-selector method (.css/.xpath/.text/.role) on one query. Use one selector style per query.

§

Timeout(Duration)

Generic operation timeout.

§

CdpTimeout

Chrome accepted a CDP command and never answered it within the call’s budget.

Distinct from ZendriverError::Cdp (Chrome answered and refused) and ZendriverError::Disconnected (the socket died): here the connection is healthy and the browser is simply wedged or slower than the budget allows. Distinct from the generic ZendriverError::Timeout because it names the CDP method, which is what makes a stuck call diagnosable.

See zendriver_transport::DEFAULT_CALL_TIMEOUT for the budget and Connection::call_raw_with_timeout / Connection::set_call_timeout to change it.

Fields

§method: String

The CDP method that was never answered (e.g. "Page.navigate").

§budget: Duration

The budget that elapsed without a reply.

§

Navigation(String)

Page navigation failed (DNS, connection refused, page crashed, etc.).

§

JsException(String)

A JS expression raised an exception during evaluation.

§

ElementStale

An element handle is stale and the auto-refresh path failed.

§

NotRefreshable

An element handle obtained from raw JS evaluation cannot be refreshed (no selector to replay).

§

NotActionable(Duration, String)

An element did not become actionable (visible + enabled + stable + hit-tested) within the gate timeout.

§

FrameNotFound(String)

Frame lookup by id / url / name failed.

§

TabNotFound(String)

Tab lookup by target_id / session_id failed.

§

Cookie(String)

A cookie operation failed (CDP refusal, malformed payload, etc.).

§

Storage(String)

A DOM storage operation failed (origin mismatch, CDP refusal, etc.).

§

HistoryNavigation(String)

Session history navigation failed (no back/forward entry).

§

Serde(Error)

JSON serialization / deserialization error at the CDP boundary.

§

Io(Error)

I/O error (file read/write, etc.).

§

PersonaSeed

A persisted persona-seed file (.zd_persona_seed inside a user_data_dir) exists but its contents are not a valid seed.

Surfaced by BrowserBuilder::resolved_persona when the seed file is corrupt or truncated. Treated as a hard error rather than silently minting a fresh random seed: for a credentialed persistent profile, the fingerprint identity silently rotating out from under the caller is a worse failure mode than refusing to launch.

Fields

§path: PathBuf

Path to the offending seed file.

§reason: String

Human-readable parse failure.

§

Stealth(Box<StealthError>)

Stealth fingerprint resolution failed.

§

Interception(Box<InterceptionError>)

Request-interception sub-crate error. Gated by feature interception.

§

Cloudflare(Box<CloudflareError>)

Cloudflare bypass sub-crate error. Gated by feature cloudflare.

§

Imperva(Box<ImpervaError>)

Imperva bypass sub-crate error. Gated by feature imperva.

§

DataDome(Box<DataDomeError>)

DataDome bypass sub-crate error. Gated by feature datadome.

§

Fetcher(Box<FetcherError>)

Chrome-for-Testing fetcher error. Gated by feature fetcher.

§

NetworkMonitor(String)

Network monitor error (setup, task failure, CDP subscription problem).

§

Browser-context HTTP request error.

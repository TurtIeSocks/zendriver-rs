Stealth configuration passed to BrowserBuilder::stealth(...).

impl StealthProfile

pub fn off() -> StealthProfile

No stealth: stock browser launch.

Use this when you want a bare-bones Chrome with none of the launch flags or CDP overrides applied — e.g. when verifying that a problem reproduces in vanilla Chrome.

use zendriver_stealth::StealthProfile;
let p = StealthProfile::off();
assert!(p.build_flags().is_empty());

pub fn native() -> StealthProfile

Launch flags + UA scrub + Emulation overrides. No JS bootstrap.

Safe against Function.prototype.toString detection (it doesn’t patch any prototype getter). The default when stealth is requested.

use zendriver_stealth::StealthProfile;
let p = StealthProfile::native();
assert!(!p.build_flags().is_empty());

pub fn spoofed() -> StealthProfile

native + Navigator-prototype JS patches. Passes the sannysoft detection battery.

Sets bypass_csp on by default so the bootstrap script can install on pages with strict CSP headers.

use zendriver_stealth::StealthProfile;
let p = StealthProfile::spoofed();
assert!(p.bypass_csp_enabled());

pub fn fingerprint(self, f: Fingerprint) -> StealthProfile

Override the auto-detected Fingerprint wholesale.

Use this when you need to pin a specific Chrome version / platform / hardware combination across runs (e.g. to keep request fingerprints stable across CI invocations).

pub fn memory_gb(self, gb: u32) -> StealthProfile

Override the reported navigator.deviceMemory (in GB).

Per the HTML Device Memory spec the value Chrome reports is one of {1, 2, 4, 8} (Chrome does not expose fractional values to JS). The resolver snaps gb to the nearest valid integer at or below it:

Input gbReported
0 or 11
2 or 32
4–74
8+8

Snap-down (not nearest-round) matches Chrome’s own behavior and avoids inflating the reported value above what the host plausibly has. Anything outside {1, 2, 4, 8} is itself a stealth tell because real browsers never report it.

pub fn cpu_count(self, n: u32) -> StealthProfile

Override the reported navigator.hardwareConcurrency (CPU count).

Clamped to 2..=32 at resolve time — values outside that range are implausibly low/high and trip simple heuristics.

pub fn chrome_version(self, major: u32) -> StealthProfile

Override the reported Chrome major version (e.g. 125).

pub fn platform(self, p: Platform) -> StealthProfile

Override the reported Platform (navigator.platform + UA OS token + UAM platform).

pub fn locale(self, l: impl Into<String>) -> StealthProfile

Override the reported locale (e.g. "en-US", "fr-FR").

Sends Emulation.setLocaleOverride and adds --lang=... to the launch flags.

pub fn languages( self, langs: impl IntoIterator<Item = String>, ) -> StealthProfile

Override the reported language list (drives navigator.languages + q-weighted Accept-Language). When unset, derived from locale.

pub fn timezone(self, tz: impl Into<String>) -> StealthProfile

Override the reported timezone via Emulation.setTimezoneOverride (IANA name, e.g. "America/Los_Angeles").

pub fn user_agent(self, ua: impl Into<String>) -> StealthProfile

Override the reported User-Agent string verbatim.

Skips the composed-from-fingerprint step — prefer platform + chrome_version unless you need an exact UA string.

pub fn bypass_csp(self, on: bool) -> StealthProfile

Toggle Page.setBypassCSP. Default true for spoofed, false for native / off.

pub fn arg(self, flag: impl Into<String>) -> StealthProfile

Add a single extra Chrome launch flag (e.g. "--proxy-server=...").

pub fn args(self, flags: impl IntoIterator<Item = String>) -> StealthProfile

Add a batch of Chrome launch flags.

pub fn resolve_fingerprint( &self, chrome_exe: &Path, ) -> Result<Fingerprint, StealthError>

Resolve the final Fingerprint: either the explicit override or an auto-detected baseline, with per-field tweaks (platform, locale, memory_gb, …) applied on top.

chrome_exe is invoked with --version to probe the Chrome major; the probe failing falls back to a baked-in default so the resolver never errors solely on Chrome being unavailable.

§Errors

Returns [StealthError::ChromeVersionDetect] when the Chrome probe fails and no override is provided, and [StealthError::SystemInfo] when total-RAM detection fails.

pub fn build_flags(&self) -> Vec<String>

Composed Chrome launch flag list: per-profile defaults plus any extras added via arg / args, with a --lang=<locale> flag injected when locale is set.

use zendriver_stealth::StealthProfile;
let flags = StealthProfile::native().locale("fr-FR").build_flags();
assert!(flags.iter().any(|f| f == "--lang=fr-FR"));

pub fn bypass_csp_enabled(&self) -> bool

Whether Page.setBypassCSP will be sent for this profile. Defaults to true for spoofed and false otherwise; the bypass_csp setter toggles it explicitly.

pub fn input_profile(&self) -> InputProfile

Returns the input-realism profile appropriate for this stealth profile. spoofed returns realistic timings; native and off return zero-overhead.

Direct-download tools — browser_download, browser_set_download_path.

browser_download ports nodriver’s page-driven fetch-and-save: it injects a main-world script that fetches the URL through the page’s own network context (cookies / referer / same-origin creds) and saves it via Chrome’s download behavior. It is fire-and-forget — it returns once the fetch is dispatched, not when the file lands. For await/inspect/save-to-path semantics use browser_expect_register { kind: download, save_to }.

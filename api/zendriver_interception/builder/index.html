InterceptBuilder — fluent rule + pattern registration.

Two-phase API:

  • Configure: chain block, block_hosts, redirect, respond, modify_request, modify_response for declarative rules, plus pattern / at_request / at_response / resource to control which CDP Fetch.RequestPattern entries are sent on Fetch.enable.
  • Activate: start spawns the actor task (T6) with the registered rules + patterns, returning an InterceptHandle for RAII teardown. Alternatively, subscribe returns a Stream<Item = PausedRequest> for the manual escape-hatch path — callers drive Chrome’s interception loop themselves.

The tab field is a borrow of SessionHandle (not the full Tab from zendriver core) — this crate must not depend on zendriver (cycle). Tab::intercept() in zendriver constructs the builder via InterceptBuilder::new(self.session()).

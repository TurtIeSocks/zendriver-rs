pub fn bootstrap_script(persona: &Persona, identity: &Fingerprint) -> String
Expand description

Build the bootstrap script for the spoofed profile.

identity supplies coherent UA-CH metadata and the real probed Chrome version; persona overrides simple identity fields (platform / hardware / locale) and drives the new fingerprint surfaces. Splitting the two keeps the UA coherent: when the persona changes platform we rebuild the UA-CH metadata against the real Chrome version instead of a stale fallback.

Order: identity IIFE first (webdriver-most-probed-first, navigator_props last), then the PRNG definition, then each persona surface.

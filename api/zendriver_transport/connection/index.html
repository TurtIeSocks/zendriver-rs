Connection — the public handle to the transport actor.

Structs§

Connection
Cheap-to-clone handle to the connection actor. All Tabs and Elements hold one of these (via Arc<...>); the actor itself runs in a separate tokio task.

Constants§

DEFAULT_CALL_TIMEOUT
Default ceiling on how long Connection::call_raw waits for Chrome to answer a single CDP command.

Functions§

connect
Connect to a Chrome DevTools WebSocket URL and spawn the actor with no observers. Convenience wrapper for connect_with_observers.
connect_with_observers
Connect to a Chrome DevTools WebSocket URL and spawn the actor with the provided TargetObserver chain. Observers fire on Target.attachedToTarget (serially, in registration order) before the actor releases the debugger.
spawn_actor
Spawn the actor on the given pre-connected WebSocket with no observers. Mainly for tests and for connect; production code uses connect.
spawn_actor_with_observers
Spawn the actor on the given pre-connected WebSocket with observers.
spawn_actor_with_observers_and_timeout
Spawn the actor with a custom observer_timeout. Exposed primarily for tests that need to assert timeout behavior without waiting on the 5 s default; production callers should prefer spawn_actor_with_observers.

1//! Cloudflare Turnstile bypass for `zendriver`.
2//!
3//! See the [Cloudflare chapter](https://turtiesocks.github.io/zendriver-rs/cloudflare.html)
4//! of the [zendriver-rs user guide](https://turtiesocks.github.io/zendriver-rs/)
5//! for end-to-end usage, timeout tuning, and detection-failure diagnostics.
6//!
7//! **Stealth recommended.** Cloudflare Turnstile is somewhat forgiving of
8//! non-stealth Chrome, but `BrowserBuilder::stealth` significantly raises
9//! the clearance success rate.
10//!
11//! Drives the Turnstile checkbox click flow:
12//!
13//! 1. Detect the Turnstile iframe via a shadow-DOM-aware walk of the page's
14//!    main world.
15//! 2. Dispatch a raw left-click at the 15% × 50% offset inside the iframe
16//!    bbox (the canonical Turnstile checkbox location).
17//! 3. Poll for either the `cf-turnstile-response` input gaining a token, or
18//!    the challenge container disappearing entirely.
19//!
20//! Most users go through `zendriver`'s `Tab::cloudflare()` (feature-gated)
21//! rather than constructing the bypass directly. The
22//! [`CloudflareBypass`] type is the underlying driver.
23//!
24//! ```no_run
25//! # async fn ex(tab: &zendriver_transport::SessionHandle)
26//! #   -> Result<(), zendriver_cloudflare::CloudflareError> {
27//! use std::time::Duration;
28//! use zendriver_cloudflare::{CloudflareBypass, ClearanceOutcome};
29//!
30//! let outcome = CloudflareBypass::new(tab)
31//!     .wait_for_clearance(Duration::from_secs(30))
32//!     .await?;
33//! match outcome {
34//!     ClearanceOutcome::TokenAcquired(token) => println!("got token: {token}"),
35//!     ClearanceOutcome::ChallengeGone => println!("challenge cleared"),
36//!     ClearanceOutcome::TimedOut { saw_challenge } => {
37//!         println!("timed out; saw_challenge = {saw_challenge}")
38//!     }
39//! }
40//! # Ok(()) }
41//! ```

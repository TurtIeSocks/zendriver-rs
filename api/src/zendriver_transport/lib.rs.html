1//! Internal CDP transport for `zendriver`: WebSocket I/O, command/response
2//! routing, event broadcast.
3//!
4//! > [!WARNING]
5//! > This crate has **no independent SemVer contract** — its surface may
6//! > rearrange in any minor release. Depend on the [`zendriver`] crate's
7//! > re-exports instead; see [`SEMVER.md`] for the policy.
8//!
9//! For a high-level walkthrough of the actor/observer model see the
10//! [Architecture chapter](https://turtiesocks.github.io/zendriver-rs/architecture.html)
11//! of the [zendriver-rs user guide](https://turtiesocks.github.io/zendriver-rs/).
12//!
13//! # What lives here
14//!
15//! - [`Connection`] — cheap, clonable handle to the actor task. All `Tab`s
16//!   and `Element`s hold one.
17//! - [`SessionHandle`] — a connection scoped to a particular CDP `sessionId`.
18//! - [`CdpCommand`] / [`CdpInbound`] / [`CdpRpcError`] / [`RawEvent`] — wire
19//!   types.
20//! - [`AccountedRawEvent`] — opt-in loss-accounted alternative to
21//!   `RawEvent`, delivered by
22//!   [`Connection::subscribe_raw_accounted`](connection::Connection::subscribe_raw_accounted);
23//!   reports lag, reconnects, and disconnects explicitly instead of silently
24//!   dropping frames.
25//! - [`TargetObserver`] — observer trait fired on `Target.attachedToTarget`
26//!   before the debugger is released; used by `zendriver-stealth` to install
27//!   patches on new pages.
28//! - [`CallError`] / [`TransportError`] — error types surfaced via
29//!   `zendriver`'s `ZendriverError::Transport` / `Cdp` variants.
30//!
31//! [`zendriver`]: https://docs.rs/zendriver
32//! [`SEMVER.md`]: https://github.com/TurtIeSocks/zendriver-rs/blob/main/SEMVER.md#internal-crates

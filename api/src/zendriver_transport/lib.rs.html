1//! Internal CDP transport for `zendriver`: WebSocket I/O, command/response
2//! routing, event broadcast.
3//!
4//! **Use via the `zendriver` crate's re-exports.** This crate is published so
5//! the workspace can compile end-to-end but its surface is not covered by the
6//! same SemVer guarantees as `zendriver`; expect minor versions to rearrange
7//! types here freely. See [`SEMVER.md`] in the repo root for the policy.
8//!
9//! # What lives here
10//!
11//! - [`Connection`] — cheap, clonable handle to the actor task. All `Tab`s
12//!   and `Element`s hold one.
13//! - [`SessionHandle`] — a connection scoped to a particular CDP `sessionId`.
14//! - [`CdpCommand`] / [`CdpInbound`] / [`CdpRpcError`] / [`RawEvent`] — wire
15//!   types.
16//! - [`TargetObserver`] — observer trait fired on `Target.attachedToTarget`
17//!   before the debugger is released; used by `zendriver-stealth` to install
18//!   patches on new pages.
19//! - [`CallError`] / [`TransportError`] — error types surfaced via
20//!   `zendriver`'s `ZendriverError::Transport` / `Cdp` variants.
21//!
22//! [`SEMVER.md`]: https://github.com/cdpdriver/zendriver-rs/blob/main/SEMVER.md

1//! Anti-detection profiles and patches for `zendriver`.
2//!
3//! Three stealth modes, exposed via [`StealthProfile`]:
4//!
5//! - [`StealthProfile::off`] — no stealth (stock Chrome launch).
6//! - [`StealthProfile::native`] — launch flags + UA scrub + CDP `Emulation`
7//!   overrides. Safe against `Function.prototype.toString` probes; the
8//!   default.
9//! - [`StealthProfile::spoofed`] — `native` plus a Navigator-prototype JS
10//!   bootstrap script that passes the [sannysoft][sannysoft] battery.
11//!
12//! Most users go through `zendriver`'s `BrowserBuilder::stealth(...)`:
13//!
14//! ```
15//! use zendriver_stealth::{Platform, StealthProfile};
16//!
17//! let profile = StealthProfile::spoofed()
18//!     .platform(Platform::MacIntel)
19//!     .locale("en-US")
20//!     .timezone("America/Los_Angeles");
21//!
22//! assert!(profile.bypass_csp_enabled());
23//! assert!(!profile.build_flags().is_empty());
24//! ```
25//!
26//! The [`StealthObserver`] type plugs into `zendriver-transport`'s observer
27//! chain and applies the resolved [`Fingerprint`] / bootstrap to every newly
28//! attached page target before Chrome releases the debugger.
29//!
30//! [sannysoft]: https://bot.sannysoft.com/
31

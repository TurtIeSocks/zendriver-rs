# Geo-IP-derived locale — design

**Issue:** [#39](https://github.com/TurtIeSocks/zendriver-rs/issues/39) — Generative
fingerprints: geo-IP-derived locale (coherent `navigator.language` /
`Accept-Language`). Follow-up split from #25; sibling of header-network #38 and a
future timezone-from-geo.

**Date:** 2026-06-03

## Problem

`Persona.locale` is a user-supplied `Option<String>` fed *verbatim* into
`Accept-Language`. Two gaps:

1. **No q-weights.** `.locale("en-US")` emits `Accept-Language: en-US` (no
   `;q=`); only the `None` fallback (`"en-US,en;q=0.9"`) carries weights.
2. **No language list.** `navigator.languages` and multi-language countries
   can't be expressed by one string. Worse, the current `navigator.languages`
   construction (`patches.rs:125`) hardcodes `"en"` as the second entry, so a
   `fr-FR` persona reports `["fr-FR","en"]` instead of the correct
   `["fr-FR","fr"]` — a latent coherence bug this work fixes.

Generating a **random** locale is a detection liability (language must match the
exit IP's apparent geo). The fix is to derive locale from a **country**, so
language coherently matches apparent location.

## Goals

- Derive a coherent `locale` + `Accept-Language` (q-weighted) +
  `navigator.languages` from an explicit **country code**.
- Use a reputable source: Unicode **CLDR** `territoryInfo.languagePopulation`
  (+ `likelySubtags` for region forms).
- Opt-in and fully overridable; never lock a value (fingerprint philosophy).
- Leave a clean seam for a later auto-resolve-from-exit-IP helper.

## Non-goals (this PR)

- **Auto IP-geo resolution (option B):** probing the exit IP through the proxy
  to discover the country. Only the `GeoResolver` *seam* ships now; the
  `IpApiResolver` + structured `ProxyUrl` + reqwest probe land in a follow-up PR
  (they drag in the unresolved proxy-config surface).
- **timezone-from-geo:** future work; will share the `geo` module.
- Emitting *all* official languages for multilingual countries (see Derivation).

## Decisions (resolved during brainstorming)

| Fork | Decision |
|------|----------|
| Resolution model | **A** (caller supplies country) is the core; **B** (auto-probe) is a seam now, impl next PR. |
| CLDR data delivery | **Vendored generated table** (committed Rust), regenerated by an offline bin + weekly workflow. |
| Sync cadence | **Weekly** cron + manual `workflow_dispatch`. |
| Sync delivery | **Open a PR**, **auto-merge enabled, gated on a sanity-guard job**. |
| Locale type shape | **Option 2** — add `Persona.languages: Option<Vec<String>>` next to `locale`. |
| Multilingual default | Dominant language only; multi-language via explicit override. |

## Architecture & placement

Dependency graph: `zendriver-stealth ← {zendriver, zendriver-fingerprints,
zendriver-mcp}`. `zendriver` does **not** depend on `zendriver-fingerprints`.
Therefore the locale primitives live in **`zendriver-stealth`**, reachable by all
consumers. Two layers:

- **Always-on (stealth):** `Persona.languages` field, pure `accept_language()`
  q-weight formatter, `resolve_languages()` helper, the navigator.languages fix.
  No new dependency, no feature gate — the observer needs the formatter
  unconditionally.
- **Feature-gated `geo` (stealth):** `Country` type, the vendored CLDR table, and
  `geo::persona(Country)`. The heavy generated table sits behind
  `--features geo` so non-geo users don't compile it.

`zendriver`, `zendriver-fingerprints`, and `zendriver-mcp` each gain a thin
`geo` feature = `["zendriver-stealth/geo"]`. The future timezone-from-geo work
slots into the same `stealth::geo` module ("shared geo module").

```
stealth::geo  (feature)   Country, CLDR table, geo::persona()   ← #39 data core
stealth::lang (always-on) accept_language(), resolve_languages()
        ▲                              ▲
   zendriver(.geo_locale)       fingerprints(generative overlay)
```

## Apply path (single source of truth)

New always-on pure functions (e.g. `zendriver-stealth/src/lang.rs`):

```rust
/// ["de-DE","de","en"] -> "de-DE,de;q=0.9,en;q=0.8"
/// q: index 0 implicit 1.0; q_i = (1.0 - 0.1*i) clamped to >= 0.1, one decimal.
/// Dedups while preserving order; empty input -> "".
pub fn accept_language(langs: &[String]) -> String;

/// Precedence: persona.languages -> fingerprint.languages -> derive(&[locale]) -> ["en-US","en"].
/// derive("fr-FR") = ["fr-FR","fr"]  — base entry is the lang subtag, NOT hardcoded "en".
pub(crate) fn resolve_languages(persona: &Persona, fp: &Fingerprint) -> Vec<String>;
```

Wire-up changes:

- `patches.rs:125` — `navigator.languages` value becomes
  `resolve_languages(persona, fingerprint)` (removes the hardcoded-`"en"` bug).
- `observer.rs:91` — `acceptLanguage` becomes
  `accept_language(&resolve_languages(persona, fingerprint))` (was
  `fingerprint.locale` verbatim). The observer must have the resolved languages
  available; populate `Fingerprint.languages` during resolve so the apply path
  reads one resolved value.
- `--lang` flag (profile.rs:342) and `Emulation.setLocaleOverride`
  (observer.rs:130) are unchanged — both stay keyed on the primary `locale`
  (which also drives `navigator.language` singular via CDP).

Coherence invariant: when set via geo, `locale == languages[0]` and the base
language is `languages[1]`.

## geo module + CLDR derivation policy (`geo` feature)

```rust
pub struct Country([u8; 2]);                 // validated ASCII alpha-2, stored uppercased
impl TryFrom<&str> for Country { /* exactly 2 ASCII letters, else Err */ }
impl std::str::FromStr for Country { /* same */ }

/// Build a Persona overlay carrying ONLY locale + languages for the country.
/// Unknown country (valid syntax, absent from table) -> Persona::default() and a tracing::warn.
pub fn persona(c: Country) -> Persona;
```

Generated table row: `country (alpha-2) -> ranked [(lang_subtag, pop_percent, official)]`,
from CLDR `territoryInfo.languagePopulation`.

Derivation (default):

- **Primary language** = highest-population language with official status
  (official / de_facto_official / official_regional); population breaks ties.
- `locale = "{lang}-{REGION}"` (e.g. `de` + `CH` -> `de-CH`).
- `languages = ["{lang}-{REGION}", "{lang}"]` (two entries).
- **No `"en"` injection.** A fresh German Chrome sends `de-DE,de`; matching real
  single-user installs is more coherent than appending English.
- **Multilingual countries** (CH/BE/CA) resolve to their single dominant
  language by default. Users opt into multiple via explicit `.languages([...])`.
  Rationale: a real user typically configures one language; reporting every
  national language is itself an unusual fingerprint.
- **Unknown / invalid country** -> no-op (empty overlay) + `tracing::warn`. Never
  error, never lock a value.

## Generator bin + sync workflow + guard

**Generator** — new non-published workspace member `crates/locale-gen`
(`publish = false`; owns its own `reqwest` + `serde` deps so the shipped
`zendriver-stealth` crate stays dependency-free):

- Fetches a **pinned** `cldr-core` (npm tarball verified by integrity hash, or a
  CLDR release tag) — **never** the `main` branch.
- Reads `territoryInfo.json` + `likelySubtags.json`.
- Emits `crates/zendriver-stealth/src/geo/table.rs` with a provenance header:
  `// CLDR <version>, generated <date>, regen: cargo run -p locale-gen`.
- Runs `cargo fmt` on the output.

**Workflow** `.github/workflows/sync-cldr-locale.yml`:

- Triggers: `schedule` weekly (Mon 06:00 UTC) + `workflow_dispatch`.
- Runs the generator; if `git diff` is non-empty, opens a PR via
  `peter-evans/create-pull-request` titled `chore: sync CLDR locale table`
  (version bump visible in the header diff).
- Enables auto-merge (`gh pr merge --auto --squash`), gated on CI + the guard.

**Sanity guard** — a required test (also part of normal CI, not just the sync
PR). Auto-merging external-sourced data is a supply-chain surface; CLDR
(`unicode-org/cldr-json`) is highly reputable and permissively licensed, so risk
is low, but the guard converts "trust the source blindly" into "trust the source
and verify invariants in CI." Assertions:

- the table parses and compiles;
- every country maps to >= 1 well-formed BCP-47 locale;
- row count within `[100, 1000]` and did not drop > 20% versus the committed
  table (catches a CLDR shape shift that guts the table);
- the CLDR version header is present and monotonically non-decreasing (no
  downgrade).

Pass -> the PR self-merges (routine ~biannual bump). Fail -> the PR stays open
for human review.

## Public API + generative + MCP wiring

- **Builder convenience** (`zendriver`, `geo` feature):
  `BrowserBuilder::geo_locale(impl TryInto<Country>)` sets a `persona_overlay`
  carrying the derived locale + languages. One line for the caller.
- **Generative** (`zendriver-fingerprints`, `geo` feature): document and support
  the overlay pattern `generator.generate(seed).overlay(stealth::geo::persona(c))`;
  add a convenience `Generator::generate_geo(seed, country)`. This fills the
  locale gap that `mapping.rs:41` leaves (browserforge has no locale node).
- **MCP** (`zendriver-mcp`, required by project policy): the stealth/lifecycle
  tool input gains `geo_country: Option<String>` mapped onto `.geo_locale`,
  behind an mcp `geo` feature. Regenerate + accept the `insta` schema snapshots.
- **public-api baseline:** regenerate (new `Country`, `geo::*`, `languages`
  field, builder method, generative convenience) and add `covered` / `excluded`
  entries to `mcp-coverage-ledger.toml`. The auto-resolve seam is `excluded`
  until the follow-up PR ships it.

## GeoResolver seam (shipped now, impl later)

```rust
// stealth::geo — empty seam, documented; concrete impls land in the follow-up PR.
#[async_trait::async_trait]
pub trait GeoResolver {
    async fn country(&self /*, proxy context added with B */) -> Option<Country>;
}
```

Both A (caller supplies `Country`) and the future B (resolver yields `Country`)
terminate in the identical `Country -> Persona` overlay, so B is purely additive
and needs no API change to A.

## Testing

- `accept_language()` unit table: empty, 1, 2, 10+ entries (q-floor at 0.1),
  duplicate collapse, ordering.
- `resolve_languages()` precedence chain + the `fr-FR -> ["fr-FR","fr"]`
  regression (the hardcoded-`"en"` fix).
- `geo::persona()`: `US -> en-US` / `["en-US","en"]`; `DE -> de-DE`; a
  multilingual country -> its dominant language; unknown country -> empty + warn.
- `Country` parse: valid, lowercase (accepted, uppercased), too long, non-alpha.
- the sanity-guard assertions (run in normal CI too).
- bootstrap snapshot covering `navigator.languages` + the q-weighted
  `Accept-Language` for a representative persona.
